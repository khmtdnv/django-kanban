# Техническое описание системы управления проектами

## 1. Управление проектами и Канбан-доска
Модуль предназначен для визуализации рабочих процессов и управления жизненным циклом задач.

* **Структура:** Проект &rarr; Канбан-доска &rarr; Настраиваемые колонки (*Бэклог, В работе, Тестирование, Готово*).
* **Карточка задачи:** Содержит название, описание, приоритет, исполнителя, теги, плановое/фактическое время, подзадачи и зависимости (блокировки).
* **Интерактив:** Смена статуса задачи простым перетаскиванием (Drag-and-Drop).

```
[Проект] 
   └── [Канбан-доска]
          ├── Бэклог       --> [Задача 1 (Подзадачи, Сроки)]
          ├── В работе     --> [Задача 2 (Исполнитель, Логи времени)]
          ├── Тестирование --> [Задача 3 (Блокировки)]
          └── Завершено
```

## 2. Система уведомлений
Обеспечивает своевременное информирование команды о критических событиях.

* **Каналы связи:** Push-уведомления в веб-интерфейсе, Email.
* **Типы триггеров:** * *По времени:* Автонапоминания о дедлайнах (настройка за Х дней/часов).
  * *По событиям:* Смена исполнителя/статуса, новые комментарии, упоминания (@user).
* **Кастомизация:** Гибкая настройка профиля уведомлений под каждого пользователя.

```
[ Событие / Дедлайн ] ──> [ Движок уведомлений ] ──┬──> [ Web Push ]
                                                  └──> [ Email ]
```

---

## 3. Аналитика и Отчетность
Модуль для оценки продуктивности команды и прогнозирования сроков.

* **Метрики:** Распределение задач по исполнителям, Lead/Cycle Time (время выполнения), количество блокеров.
* **Прогнозирование:** Оценка сроков проекта на основе исторических данных.
* **Экспорт и шедулинг:** Выгрузка в PDF/CSV, настройка регулярной рассылки отчетов на почту (стейкхолдерам).

```
[ Данные системы ] ──> [ Модуль аналитики ] ──┬──> Экспорт (PDF / CSV)
                                              └──> Регулярная Email-рассылка
```

---

## 4. REST API
Интерфейс для бесшовной интеграции платформы с внешним корпоративным контуром.

* **Стандарты:** REST (GET, POST, PUT, DELETE), спецификация OpenAPI (Swagger).
* **Безопасность:** Аутентификация по JWT-токенам, ролевая модель доступа (RBAC).
* **Возможности:** Полное программное управление задачами, проектами, пользователями и выгрузка аналитики.
* **Сценарии интеграции:** CI/CD конвейеры, внешние тайм-трекеры, ERP и корпоративные порталы.

```
[ Внешние сервисы ] (CI/CD / ERP)
       │ 
       ▼ (JWT Token Auth)
[ REST API Gateway (Swagger / OpenAPI) ]
       │
       ├──> [ Управление задачами и проектами ]
       ├──> [ Извлечение аналитики ]
       └──> [ Администрирование пользователей ]
```
